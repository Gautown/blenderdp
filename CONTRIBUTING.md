## 行为准则

在参与本项目之前，请务必阅读并遵守我们的[行为准则](CODE_OF_CONDUCT.md)。我们致力于为所有人提供一个安全、友好的环境。

## 如何贡献

### 报告 Bug

如果您发现了 bug，请在 GitHub Issues 中搜索是否已有相关的报告。如果没有，请创建一个新的 Issue 并包含以下信息：

- 清晰且描述性的标题
- 详细的步骤说明如何重现问题
- 期望的行为和实际观察到的行为
- 如果可能的话，提供截图或 GIF 动画
- 您的环境信息（操作系统、浏览器版本等）

### 提出功能请求

如果您有一个功能建议，请先在 GitHub Issues 中搜索是否已有类似的请求。如果没有，请创建一个新的 Issue 并包含以下信息：

- 清晰且描述性的标题
- 详细的功能描述
- 解释为什么这个功能对项目有用
- 如果可能的话，提供示例代码或伪代码

### 贡献代码

如果您想直接贡献代码，请遵循以下步骤：

1. Fork 本项目
2. 创建一个新的分支 (`git checkout -b feature/your-feature-name`)
3. 进行您的修改
4. 确保您的代码符合项目的代码规范
5. 添加或更新相关文档
6. 提交您的修改 (`git commit -am 'Add some feature'`)
7. 推送到您的分支 (`git push origin feature/your-feature-name`)
8. 创建一个新的 Pull Request

## 开发环境搭建

要设置开发环境，请按照以下步骤操作：

1. 克隆您的 Fork 仓库：
   ```bash
   git clone https://github.com/your-username/blenderdp.git
   ```

2. 进入项目目录：
   ```bash
   cd blenderdp
   ```

3. 安装依赖：
   ```bash
   npm install
   ```

4. 启动开发服务器：
   ```bash
   npm run dev
   ```

5. 在浏览器中访问 `http://localhost:4321` 查看您的更改

## 代码规范

请确保您的代码遵循以下规范：

- 使用一致的缩进（2个空格）
- 遵循现有的代码风格
- 为您的代码添加适当的注释
- 确保所有测试通过（如果有测试的话）

## 提交 Pull Request

在提交 Pull Request 之前，请确保：

1. 您的代码是最新的（从主分支拉取最新更改）
2. 您的代码没有语法错误
3. 您已添加了适当的文档（如果需要的话）
4. 您的提交信息清晰且具有描述性

Pull Request 应该：

- 有一个清晰且描述性的标题
- 包含对所做更改的详细说明
- 引用相关的 Issue（如果有的话）
- 保持专注：每个 PR 应该只解决一个问题或添加一个功能
